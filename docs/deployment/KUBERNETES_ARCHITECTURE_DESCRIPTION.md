# Архитектура мессенджера для Kubernetes

## Обзор

Архитектура мессенджера для Kubernetes развертывается как Helm-чарт с несколькими компонентами:

- **PostgreSQL StatefulSet** - кластер баз данных с персистентным хранилищем для надежного хранения данных
- **Redis StatefulSet** - кластер Redis с персистентным хранилищем для кэширования и pub/sub функций
- **MinIO StatefulSet** - объектное хранилище файлов, совместимое с S3, с персистентным хранилищем
- **Messenger Server Deployment** - основной сервер приложения с возможностью автомасштабирования
- **Messenger Service** - сервис для внутренней маршрутизации трафика внутри кластера
- **Ingress Controller** - маршрутизация внешнего трафика ко внутренним сервисам
- **Load Balancer** - распределение входящего трафика между инстансами приложения
- **Helm Chart** - шаблоны и конфигурации для деплоя всех компонентов

## Связи между компонентами

- Messenger Server Deployment подключается к PostgreSQL StatefulSet для хранения и получения данных
- Messenger Server Deployment использует Redis StatefulSet для кэширования и управления сессиями
- Messenger Server Deployment взаимодействует с MinIO StatefulSet для хранения и получения файлов
- Messenger Server Deployment взаимодействует с ConfigMap и Secret для получения конфигурации и секретов
- Messenger Service обеспечивает внутреннюю маршрутизацию трафика в кластере
- Ingress Controller направляет внешний трафик к Messenger Service
- Load Balancer распределяет трафик между инстансами приложения
- Клиентские приложения подключаются к мессенджеру через Load Balancer

## Инфраструктурные особенности

- Используются StatefulSet для компонентов, требующих персистентного хранилища
- Используется Deployment для масштабируемого серверного приложения
- Используются PersistentVolume для обеспечения персистентности данных
- Helm-чарт предоставляет шаблоны для всех компонентов
- Поддерживается автомасштабирование для основного серверного приложения
- Архитектура предназначена для продакшн-развертывания с высокой доступностью
