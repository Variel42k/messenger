# Анализ проекта Messenger

## Обзор проекта

Проект Messenger представляет собой полнофункциональную платформу для обмена сообщениями в реальном времени с серверной частью на Java Spring Boot и клиентской частью на JavaFX. Проект поддерживает обмен сообщениями в реальном времени, обмен файлами и аутентификацию пользователей.

## Архитектура

- **Серверная часть**: Java 17 + Spring Boot 3.x с WebFlux (реактивное программирование)
- **Клиентская часть**: JavaFX десктопное приложение
- **База данных**: PostgreSQL для основного хранения данных
- **Кэш**: Redis для отслеживания онлайн-статусов, распределенных блокировок, публикации/подписки
- **Хранилище файлов**: RustFS (совместимое с S3) для вложений
- **Реальное время**: WebSocket с протоколом STOMP
- **Аутентификация**: JWT (токены доступа и обновления) с ролевым доступом

## Особенности

- Обмен сообщениями в реальном времени через WebSocket/STOMP
- Аутентификация на основе JWT с циклом обновления токенов
- Вложения файлов с потоковой передачей загрузки/скачивания
- Отслеживание онлайн-статуса
- Подтверждения доставки сообщений
- Ролевой контроль доступа (пользователь/администратор)
- Распределенная архитектура с Redis pub/sub

## Выводы

1. **Клиентская часть** - присутствует (JavaFX приложение)
2. **Web-часть** - отсутствует (только десктопное JavaFX приложение)
3. **Серверная часть** - полностью реализована с поддержкой всех необходимых функций
4. **Инфраструктура** - настроена с использованием Docker Compose для PostgreSQL, Redis и RustFS

## Рекомендации

Для добавления web-части потребуется создать отдельное веб-приложение (например, на React, Vue или Angular) с аналогичным функционалом, как у JavaFX клиента.
