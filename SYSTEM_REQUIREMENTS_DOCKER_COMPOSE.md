# Системные требования для развертывания с использованием Docker Compose

## Минимальные системные требования

### Операционная система
- Linux (Ubuntu 18.04+, CentOS 7+, Debian 10+)
- Windows 10/11 (с WSL2 для лучшей производительности)
- macOS 10.15+

### Оборудование
- **Процессор**: 2 ядра или более
- **Оперативная память**: 4 ГБ RAM (рекомендуется 8 ГБ)
- **Свободное место на диске**: 10 ГБ

### Программное обеспечение
- **Docker**: версия 20.10 или выше
- **Docker Compose**: версия 2.0 или выше
- **Java 17** (для локальной сборки и разработки)
- **Maven 3.8+** (для локальной сборки и разработки)

## Компоненты и их требования

### PostgreSQL (v15)
- Минимум 1 ГБ RAM при работе
- Приблизительно 1-2 ГБ свободного места на диске для данных

### Redis (v7-alpine)
- Минимум 256 МБ RAM
- Мало места на диске требуется (менее 100 МБ)

### RustFS
- Минимум 512 МБ RAM
- 2-5 ГБ свободного места на диске в зависимости от объема файлов пользователей

### Сервер приложения
- Минимум 1 ГБ RAM (рекомендуется 2 ГБ)
- 512 МБ свободного места на диске

## Сетевые требования

- Открытые порты: 8080 (сервер), 5432 (PostgreSQL), 6379 (Redis), 9000 (MinIO), 9001 (консоль MinIO)
- Доступ к интернету для скачивания Docker-образов

## Рекомендуемые параметры Docker

### Docker daemon
```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "default-ulimits": {
    "nofile": {
      "Hard": 6400,
      "Name": "nofile",
      "Soft": 64000
    }
  }
}
```

## Рекомендуемые настройки производительности

### Docker Compose
- Использование named volumes вместо bind mounts для производительности
- Настройка лимитов ресурсов:
```yaml
services:
  server:
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 2G
        reservations:
          cpus: '0.5'
          memory: 1G
```

### Рекомендации по размещению
- Для продакшена рекомендуется использовать выделенный сервер или виртуальную машину
- SSD-накопители для лучшей производительности базы данных

## Требования к безопасности

- Обновленная операционная система с установленными последними патчами
- Настроенный брандмауэр
- Использование безопасных паролей для всех сервисов
- Регулярные обновления Docker-образов

## Мониторинг и обслуживание

- Минимум 10% свободного места на диске для стабильной работы
- Регулярное резервное копирование данных PostgreSQL и RustFS
- Мониторинг использования ресурсов
