# Архитектура мессенджера для Docker Compose

## Обзор

Архитектура мессенджера для Docker Compose состоит из нескольких контейнеров, работающих в одной сети. Основные компоненты:

- **PostgreSQL** - основная реляционная база данных для хранения информации о пользователях, чатах и сообщениях
- **Redis** - кэш и хранилище сессий, используется для отслеживания онлайн-статусов пользователей
- **MinIO** - объектное хранилище файлов, совместимое с S3, используется для хранения медиафайлов
- **Messenger Server** - основной сервер приложения, обеспечивающий бизнес-логику и API
- **JavaFX Client** - десктопное клиентское приложение

## Связи между компонентами

- Messenger Server подключается к PostgreSQL для хранения и получения данных
- Messenger Server использует Redis для кэширования и управления сессиями
- Messenger Server взаимодействует с MinIO для хранения и получения файлов
- JavaFX Client соединяется с Messenger Server через REST API и WebSocket протоколы

## Инфраструктурные особенности

- Все контейнеры работают в одной Docker сети (messenger-network)
- Для хранения данных используются именованные тома (volumes) для обеспечения персистентности
- Используется init.sql для начальной инициализации базы данных
- Архитектура предназначена для локального запуска и разработки
