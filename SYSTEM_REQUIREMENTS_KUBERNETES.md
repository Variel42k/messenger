# Системные требования для развертывания в Kubernetes с использованием Helm

## Минимальные системные требования

### Kubernetes кластер
- **Версия Kubernetes**: 1.20 или выше
- **Количество узлов**: минимум 1 узел (рекомендуется 3 узла для отказоустойчивости)
- **Операционная система узлов**: Linux (Ubuntu 18.04+, CentOS 7+, RHEL 7+)
- **Архитектура**: x86_64

### Ресурсы узлов
- **Процессор**: 2 vCPU на узел (рекомендуется 4 vCPU)
- **Оперативная память**: 4 ГБ RAM на узел (рекомендуется 8 ГБ)
- **Свободное место на диске**: 20 ГБ на узел

### Инструменты
- **Helm**: версия 3.0 или выше
- **kubectl**: совместим с версией кластера
- **Доступ к реестру контейнеров** для получения образов

## Компоненты и их требования

### CoreDNS
- Минимум 128 МБ RAM
- Минимум 100 милиядер CPU

### etcd (для управления состоянием кластера)
- Минимум 2 ГБ RAM (рекомендуется 8 ГБ для больших кластеров)
- Высокоскоростной диск (SSD рекомендуется)

### API сервер
- Минимум 512 МБ RAM
- Минимум 200 милиядер CPU

## Требования для приложения Messenger

### Ресурсы пода сервера
- **CPU**: минимально 250 милиядер, ограничение 500 милиядер (рекомендуемо: 1000 милиядер/2000 милиядер)
- **Память**: минимально 512 МБ, ограничение 1 ГБ (рекомендуемо: 1 ГБ/2 ГБ)

### Ресурсы PostgreSQL (Helm зависимость)
- **CPU**: минимально 250 милиядер, ограничение 1000 милиядер
- **Память**: минимально 256 МБ, ограничение 1 ГБ
- **Хранилище**: 8 ГБ (Persistent Volume)

### Ресурсы Redis (Helm зависимость)
- **CPU**: минимально 100 милиядер, ограничение 250 милиядер
- **Память**: минимально 128 МБ, ограничение 256 МБ
- **Хранилище**: 8 ГБ (Persistent Volume)

### Ресурсы MinIO (Helm зависимость)
- **CPU**: минимально 250 милиядер, ограничение 1000 милиядер
- **Память**: минимально 512 МБ, ограничение 1 ГБ
- **Хранилище**: 10 ГБ (Persistent Volume)

## Сетевые требования

### Внутренние порты
- **API сервер**: 6443
- **NodePort сервисы**: 30000-32767
- **Приложение**: 80
- **PostgreSQL**: 5432
- **Redis**: 6379
- **MinIO**: 9000

### Внешние зависимости
- Доступ к Docker Hub или частному репозиторию контейнеров
- Доступ к репозиториям Helm (bitnami, и др.)

## Хранилище

### Типы хранилища
- **Persistent Volumes** с поддержкой ReadWriteOnce (RWO)
- Для продакшена рекомендуется использовать ReadWriteMany (RWX) для MinIO

### Классы хранилища
- Наличие настроенных StorageClass в кластере
- Рекомендуется использовать SSD для базы данных

## Безопасность

### RBAC
- Разрешения для Helm доустановки ресурсов
- ServiceAccount с минимальными необходимыми привилегиями

### Сетевая безопасность
- Network Policies (опционально, но рекомендуется)
- TLS сертификаты для ingress (если используется)

## Мониторинг и логирование

### Рекомендуемые компоненты
- Prometheus для сбора метрик
- Grafana для визуализации
- Fluentd или аналог для сбора логов
- AlertManager для оповещений

## Резервное копирование

### Компоненты для резервного копирования
- Velero для резервного копирования кластера
- pgBackRest или аналог для PostgreSQL
- Встроенные механизмы MinIO для резервного копирования

## Рекомендации для продакшена

### Масштабирование
- Использование Horizontal Pod Autoscaler
- Настройка нескольких реплик (replicaCount > 1)
- Настройка LoadBalancer сервисов

### Высокая доступность
- Развертывание в нескольких зонах доступности
- Настройка отказоустойчивости для etcd
- Использование managed кластеров (EKS, AKS, GKE)
